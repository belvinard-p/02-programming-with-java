## ðŸ›¡ï¸ **Module 2: Error Handling (Defensive Programming)**

### **Why Error Handling?**
- Code may not work as intended due to **errors**
- **Defensive programming** anticipates and manages these situations

### **Exception Types**
| Type | Description |
|------|-------------|
| **Checked Exceptions** | Verified at compile time (e.g., IOException) |
| **Unchecked Exceptions** | Occur at runtime (e.g., NullPointerException) |

### **Management Techniques**
- **Try-catch blocks**: Capturing and processing exceptions
- **Preventive measures**: Avoiding common errors
- **Throwing exceptions**: Propagating exceptions when necessary

### **Skills Developed**
- Identify different types of Java errors
- Implement appropriate try-catch blocks
- Apply defensive programming strategies

### **Hands-on Practice**
- Applying error handling techniques in a lab environment
